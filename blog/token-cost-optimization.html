一、Token 成本为什么这么高？

很多团队在使用 AI 服务时，往往只关注模型效果，忽略了成本控制。等到账单出来才发现问题严重。让我们看看最常见的几个"烧钱"原因：

1. 没有缓存机制

这是最常见的浪费。相同或相似的请求重复发送，每次都消耗新的 Token。

⚠️ 典型场景：用户问"今天天气"，AI 回答后，用户刷新页面再问一次，又消耗了一遍 Token。

2. 温度参数设置不当

温度（temperature）参数控制输出的随机性。设置太高会导致输出冗长、不稳定，无形中增加了 Token 消耗。

温度值 特点 Token 消耗 适用场景
0.1 - 0.3 输出稳定、简洁 较低 代码生成、事实问答
0.4 - 0.7 平衡创意与稳定 中等 对话、写作
0.8 - 1.0 创意丰富、不稳定 较高 创意写作、头脑风暴

3. 提示词冗余

很多开发者在系统提示里写了一大堆内容，有些甚至是重复的。每次请求都要发送这些冗余信息。

// ❌ 冗余提示词示例
你是一个专业的AI助手。
你非常专业。
你有专业的知识。
你的回答要专业。
（后面还有很多重复的"专业"...）

// ✅ 优化后的提示词
你是一个专业的AI助手，提供准确、简洁的回答。

4. 没有最大 Token 限制

单次请求输出无限长，遇到"话唠"模型，一次请求可能消耗几百甚至上千 Token。

5. 模型选择不当

简单问题用复杂模型，比如用 GPT-4 做简单分类任务，完全可以用更便宜的 GPT-3.5。

二、Token 成本优化实战

方案一：智能缓存

实现相似请求的缓存复用，可以大幅减少重复请求。

// 简单的缓存实现示例
const cache = new Map();

async function cachedChat(prompt) {
    const key = hashPrompt(prompt);
    
    if (cache.has(key)) {
        return cache.get(key); // 命中缓存，不消耗 Token
    }
    
    const response = await openai.chat.completions.create({
        model: "gpt-3.5-turbo",
        messages: [{ role: "user", content: prompt }]
    });
    
    cache.set(key, response);
    return response;
}
✅ 效果：某客户通过缓存机制，将 Token 消耗降低了 25-30%。

方案二：参数优化

根据场景选择合适的参数：

场景 推荐温度 推荐 max_tokens
代码生成 0.1 - 0.3 500 - 1000
事实问答 0.1 - 0.2 200 - 500
日常对话 0.5 - 0.7 300 - 800
创意写作 0.7 - 0.9 1000 - 2000

方案三：提示词精简

好的提示词应该简洁明了：

// ❌ 冗长版（每次多消耗 100+ Token）
你是一个非常专业的客服助手，你要用最专业的方式回答用户的问题。
你要确保你的回答是准确的、专业的、有帮助的。
你要注意你的语气要友好但专业。
你要记住你代表的是公司的形象。

// ✅ 精简版（每次节省 80+ Token）
你是客服助手。回答准确、友好、专业。

方案四：智能模型路由

根据任务复杂度自动选择模型：

async function smartChat(prompt) {
    const complexity = analyzeComplexity(prompt);
    
    if (complexity === 'simple') {
        // 简单问题用便宜的模型
        return await chat(prompt, 'gpt-3.5-turbo');
    } else if (complexity === 'medium') {
        // 中等复杂度用 GPT-3.5 turbo
        return await chat(prompt, 'gpt-3.5-turbo');
    } else {
        // 复杂问题才用 GPT-4
        return await chat(prompt, 'gpt-4');
    }
}

function analyzeComplexity(prompt) {
    // 简单的复杂度判断逻辑
    const keywords = {
        simple: ['天气', '时间', '计算', '翻译', '定义'],
        complex: ['分析', '推理', '创作', '规划', '设计']
    };
    
    for (const word of keywords.simple) {
        if (prompt.includes(word)) return 'simple';
    }
    
    for (const word of keywords.complex) {
        if (prompt.includes(word)) return 'complex';
    }
    
    return 'medium';
}

四、Token 成本自检清单

立即检查你的 AI 项目，看看有没有这些问题：

  1. ✅ 是否实现了请求缓存？（相似请求复用结果）
  2. ✅ 温度参数是否合理？（大多数场景 0.5-0.7 足够）
  3. ✅ 是否设置了 max_tokens 限制？
  4. ✅ 提示词是否精简？（去掉冗余内容）
  5. ✅ 是否有模型选择策略？（简单问题用便宜模型）
  6. ✅ 是否有成本监控？（知道每天花了多少钱）
  7. ✅ 是否有异常告警？（成本突然上涨能及时发现）
